    BUSTER¶

    BUSTER is a structure refinement program written by Global Phasing Limited. Global Phasing provide extensive documentation on the use of the program:

    • BUSTER homepage (structure refinement) documentation

    • BUSTER documentation

    • BUSTER FAQ

    Input¶

    The CCP4i2 interface to BUSTER comprises input data and the most common options.

    The mandatory input data is a set of reflections, a FreeR set and an atom model. Optionally a ditionanary of ligand restraints may be provided.

    Main Inputs

    Atomic Model – File containing model coordinates and other parameters to be refined - typically PDB/mmCIF format.

    Reflections – File containing structure factor amplitudes/intensities and associated uncertainties (F/SigF or I/SigI) - typically MTZ format.

    Free R Set – Free R set corresponding to the reflection data provided. If the reflection data file (e.g. MTZ file) contains a column specifying the free set then re-supply the data file here. Otherwise, a new free R set data object can be generated using the “Generate a Free R set” task (in the “Data reduction” section).

    The reflections and Free R will typically be the output of a Data reduction, Import merged data, etc. task.

    Optional Additional Inputs

    Ligand description – Restraint dictionary (mmCIF file) representing ideal ligand geometry. This file specifies restraints to be used during refinement. Such restraints must be provided for any non-standard ligand/residue/heterogen in the input model. To generate restraints using Acedrg, see the “Make Ligand” task in the “Model building and refinement” section.

    Mandatory options

    Number of refinement cycles

    The number of BIG (refinement/water building/bulk solvent mask update/weight adjustment) and SMALL (refinement iterations during each BIG cycle). These are 5 and 100 by default respectively. A fuller description is in the BUSTER command line documentation .

    Water solvent treatment

    BUSTER’s solvent treatment must be specified. There are 3 options:

    • No automatic update

    • Switch on water updates for every big cycle

    • Switch on water solvent treatment for big cycle n onwards

    These options and advice on how to use them can be found in the BUSTER waters documentation

    Additional options

    • Automatically detect NCS (non-crystallographic symmetry) and impose LSSR (local structure similarity restraints) - This turns on the -autoncs option in BUSTER.

    • Turn on rigid body refinement for the first big cycle

    • Carry out TLS (Translation/Libration/Screw) refinement using a simple model (one domain per chain) - Use the TLSBasic option with a single TLS group per macro-molecular chain.

    Results¶

    The BUSTER report has summary showing final R/RFree, a plot of R and RFRee versus refinement cycle, BUSTER’s summary.png and summary_LL.png plots, and the usual list of input and output data.
